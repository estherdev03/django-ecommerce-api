# Django DRF E-Commerce Product Inventory API

A RESTful product inventory API built with Django and Django REST Framework. This backend service provides a structured catalog system with hierarchical categories, product lines, attributes, and image management — designed to power e-commerce storefronts.

---

## Features

- **Hierarchical Categories** — Tree-structured categories using `django-mptt`, supporting unlimited nesting
- **Product & Product Lines** — A product can have multiple lines (e.g. variants by size/color), each with its own SKU, price, stock, and weight
- **Flexible Attribute System** — Define custom attributes (e.g. color, size) and assign values per product or product line
- **Product Images** — Multiple ordered images per product line with alt-text support
- **Product Types** — Group products by type and associate expected attributes to each type
- **Active/Inactive Filtering** — All major models support `is_active` flags with a custom `ActiveQueryset` manager
- **Auto-ordered Fields** — Custom `OrderField` ensures sequential ordering scoped to parent records
- **OpenAPI Docs** — Auto-generated Swagger UI via `drf-spectacular` at `/api/schema/docs/`
- **Render Deployment Ready** — Includes `render.yaml` for one-click deployment to [Render](https://render.com)

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/category/` | List all active categories |
| `GET` | `/api/product/{slug}/` | Retrieve a single product by slug |
| `GET` | `/api/product/category/{slug}/` | List all products under a category |
| `GET` | `/api/schema/` | Download OpenAPI schema |
| `GET` | `/api/schema/docs/` | Interactive Swagger UI |

## Getting Started

### Prerequisites

- Python 3.11+
- pip

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/estherdev03/django-ecommerce-api.git
   cd django-drf-ecommerce
   ```

2. **Create and activate a virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Set up environment variables**

   Create a `.env` file in the project root:
   ```env
   SECRET_KEY=your-secret-key-here
   ```

5. **Apply migrations**
   ```bash
   python manage.py migrate
   ```

6. **Run the development server**
   ```bash
   python manage.py runserver
   ```

   The API will be available at `https://product-inventory-api-51eo.onrender.com/api/schema/docs/`

## Deployment

This project is configured for deployment on [Render](https://render.com) using the included `render.yaml`.

The configuration will:
- Install dependencies and collect static files
- Run database migrations automatically on deploy
- Serve with Gunicorn
- Use a managed PostgreSQL database

Set the following environment variables in your Render dashboard (or they are auto-generated by `render.yaml`):
- `SECRET_KEY`
- `DATABASE_URL`
- `DJANGO_SETTINGS_MODULE` → `ecommerce.settings.production`

---

## Settings

The project uses environment-based settings split across three files:

- `settings/base.py` — common settings shared across all environments
- `settings/local.py` — SQLite database, debug toolbar, etc.
- `settings/production.py` — PostgreSQL, WhiteNoise, security hardening

Set the active settings module via:
```bash
export DJANGO_SETTINGS_MODULE=ecommerce.settings.local
```
